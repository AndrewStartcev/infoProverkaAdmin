🪟 Модальные окна (Попапы)

Все попапы из chunk/popup.html. Открываются через атрибут data-popup="#id" на любом элементе.

#getTarifs
Тарифный план

Большой попап (max-width: 968px, класс --big). Подробное описание тарифа: 5₽ / 5 дней пробного периода, далее 399₽/месяц или 99₽/неделю. Инструкция по отмене подписки. Ссылка на форму поддержки.

--bigИнформационный

Форма поддержки

Стандартный попап (max-width: 640px). Содержит полную форму обращения: ФИО, телефон, email, тема, сообщение, два чекбокса. После отправки → открывает #formSuccess.

Попап-уведомление. Иконка успеха, заголовок «Запрос отправлен», подпись «Спасибо, что обратились к нам за помощью…», кнопка возврата на главную.

⚡ JavaScript

Функционал из src/assets/js/main.js.

🪟
Класс Popup

Управление всеми модальными окнами. Открытие через data-popup, закрытие через .popup__close, data-close, Escape или клик на оверлей.

class Popup { _init() // слушатели событий _lock() // блок прокрутки body _unlock() // разблок прокрутки open(id) // открыть по ID closeAll() // закрыть все showSuccess() // показать #formSuccess }
☰
Мобильное меню

Бургер-кнопка открывает боковое меню. Закрытие: кнопка «×», клик на оверлей, Escape. Блокирует прокрутку страницы.

// Классы при открытии: menu.classList.add('is-open') overlay.classList.add('is-visible') body.style.overflow = 'hidden'
Все формы при сабмите должны вызывать popup.showSuccess() для показа попапа успеха #formSuccess, либо обрабатываться бекендом с редиректом.

Важные замечания по вёрстке
!
Атрибут name у полей
Некоторые поля форм используют name="surname" и name="patronymic" вместо семантических имён. При интеграции с бекендом необходимо исправить name на корректные: phone, password, fio, email, subject, message, tracking_number и т.д.
!
Action форм пустой
Все формы имеют action="". Необходимо задать URL API-эндпоинтов при интеграции.
i
Попап после успешной отправки
Форма поддержки при успехе должна открывать попап #formSuccess через JS: popup.showSuccess().
i
Авторизация и редирект
После успешного входа (auth.html) — редирект на dashboard.html. После регистрации — редирект на auth.html или premium.html.
i
Мониторинг виджет
Toggle-свитч на дашборде должен отправлять AJAX-запрос для сохранения состояния. Поле «Последнее обновление» должно заполняться с бекенда.
✓
Адаптивность
Вёрстка полностью адаптивна. Брейкпоинты в ui-kit/media.scss. Мобильное меню работает через JS.
